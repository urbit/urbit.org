# Urbit Core Glossary

This reference defines common terms used by the public Urbit skill collection. Prefer upstream documentation for authoritative detail.

- **Ship** — an Urbit node identity such as a galaxy, star, planet, moon, or comet.
- **Pier** — the local directory that stores a ship's runtime state.
- **Vere** — the Urbit runtime implementation commonly used to run ships.
- **Arvo** — the Urbit operating system kernel.
- **Desk** — a Clay namespace containing source, marks, agents, generators, and other files.
- **Clay** — the Urbit filesystem vane.
- **Gall** — the application vane for long-running agents.
- **Mark** — a typed data conversion/validation module, commonly stored under `/mar`.
- **Generator** — a Hoon program commonly invoked from Dojo or via Khan.
- **Dojo** — the interactive Urbit shell.
- **Eyre** — the HTTP vane.
- **Ames** — the networking vane.
- **Kelvin** — Urbit's decrementing kernel version number.
- **Comet** — an anonymous Urbit identity useful for low-stakes testing and onboarding.
- **Fake ship** — a local non-live ship, usually used for development or tests.
